You’re probably wondering, “What’s considered good design??”. “Good/Bad” are very subjective terms, but there are some elements that we look/don’t look for. In general, we want to promote high quality templates on RapidCMS.com

When we approach template reviews, we are operating from the standpoint of RapidCMS and potential RapidCMS customers who may purchase your template in our template marketplace. Through customer interviews, thousands of customer comments, and thousands of template reviews, we’ve discovered that how a template is perceived is based on how flexible / dynamic / customizable it is.

Files and Folder Structure

The files and folder structure of a RapidCMS template is relatively simple. There are a few gotchas and best-practices worth noting but fundamentally a template is comprised of HTML, CSS, Javascript, and some media assets, along with a mapping layer from your template to the RapidCMS Website Builder

Template Folder Structure Description

rapidcms-template Required

Every template must have a rapidcms-template folder at its root.


css Required

style.css must be saved locally and included with the template inside the css folder. style-color1.css - style-color2.css - style-color3.css - style-color4.css - style-color5.css (colors are needed to change overall color pattern)


fonts Optional

We recommend saving and including any local fonts using a fonts folder.
Google Fonts - is recommended for all fonts


images Optional

Important Images included in the images/ directory that are not used within the template will cause errors when the template is uploaded.


images Best Practice

Use web-optimized images. A tool like Smush.it can be quite handy. Also recommend putting them in an img folder.


js Required

All scripts are Required to be inside the js folder.
Important HTML5Shiv must be included and linked within index.html.


js Best Practice

Put common or custom Javascript code in a single file in the js/ directory (i.e. js/functions.js).


meta Required

The meta/ directory must exist and contain template.json and the form object needs to be within template.json or the template will fail to upload.


meta/template.json Required

The style object needs to be defined in meta/template.json, otherwise you’ll see a validation error


index.html Required

meta charset="utf-8" must exist in the head of index.html

checkout.html Required


thankyou.html Required


Important note:
Currently, your script tags must be located within your head tag.

Assets\js\components

A folder with the global js files that use in the templates
For example :

Datepicker.js
Validator.js
Customcheckboxes.js
Featherlight.js

Metadata / Instructions

The template.json defines the types of elements users will interact with, such as a container, an image, text, editable images and text etc… But before we explain how it works, please see the Extremely Important!!! note below:

Extremely Important!!!

meta/template.json

This is so important, we are going to say it a couple of times. And we’re going to do it before we even tell you what element ID’s or our template.json are. AND we’re going to include this animated gif to show you what might happen if you change element data-lead-id in your html or the id within your template.json file: Click here to download JSON file



editingIDs.gif

When RapidCMS.com customers buy your template, and edit it within our RapidCMS Website Builder, the changes they make are based on the ID’s you define in the template.json file. If you change those ID’s, you will break the mapping from their published page to your template. Highly recommend you read our best practices for IDs.


To sum it all up:
NEVER change data-lead-id="container-1" to data-lead-id="container-1a" This will break pages using your template.

After reviewing the notes above carefully, please toggle to the json tab to view the template.json file.

Available Properties

id Required

A unique identifier for the template.
Example(s): “123456”, “LP-1234”, “zBxVL”, etc.


name Required

The name of the template. This can be changed within the RapidCMS interface, if needed.
An elements name attribute cannot be the same as its id attribute
Example(s): “Video Squeeze Page”, Facebook Promo Page", etc.


mobile Required

Is the template mobile responsive?
Example(s): true, false (default)


notes Required

A brief description of the template or a comment concerning any changes (for updates).
Example(s): “Initial creation!”


syntax Required

The syntax for the template. Currently the system only accepts a syntax value of 4.
Example(s): 4


version Required

The current version number of the template. Each subsequent update to a template must increment the version number in order to upload properly.
Example(s): 1.0, 2.19


form Required

An array of objects that customize the LeadBox™ appearances. As of syntax 4.0, the form object replaces the /meta/form.html method to customize the LeadBox™. If you still have form.html, you’ll see a Deprecated form.html warning.
Example(s): See Deprecated form.html.


elements Required

An array of objects that informs the RapidCMS editor of elements that are editable. Requires at least one object to upload warningfully.
Example(s): See each element for more information on the available object properties.


style Required

An object containing an array of replacement objects (read: instructions).
Example(s): See the styles section for more information on the available properties of a replacement object.


variables Optional

An object that connects- an element with configuration data and any Javascript methods that might utilize the RapidCMS_input_data object.
Example(s): See the Facebook Share Button for an example.


migrate Optional

An object that defines mapping between old element ID’s & new element ID’s. This ensures that any changes made to element ID’s will remain backwards compatible with previous versions.
Example(s): See the Migrating Element IDs for further detail.

Styles

A breakdown of the tools available for color customizations (styles). The “style” property within meta/template.json currently supports only one property - “replacements”. The “replacements” property expects an array of objects.

You use the replacements array to define the colors that are editable within your template. Generally speaking, you should expose each of the colors you define. Currently, the page builder does a global find / replace within your stylesheet. Often times, template authors will just define a color palette for their template (primary, secondary, accent, light, dark, gray). This results in fewer colors that can be edited, but provides consistent theming throughout the template. Other times, authors will define colors based on specific elements (header background, header text, section 1 background, etc). In this case, if you want to expose multiple replacements for the color white, for example, you’ll have to create slightly different variants (shades), as seen in the json to the right.

Styles Properties

type Required

The type of style component to render in the editor.
Example(s): “Color”.


name Required

The label of the style to change under the “Styles” section of the editor.
Example(s): “Background Color”, “Main Font Color”, “Link Color”.


find Required

The value to “find” and subsequently replace. Note, because this operation is a find/replace, using a common color in css/style.css for many items will result in a larger-than-expected replacement.
Example(s): Setting “find” to “#CCCCCC” will change anything using that color, regardless of the cosmetic intent. It’s strongly recommended to use slight variations for more granular control (i.e. “#CCCCCD”, “#CCCCCB”, etc.)

Elements

Containers are elements within a template that can be hidden. Technically speaking any element in a template that can be edited, can be hidden or deleted by RapidCMS customers. The “container” type meta/template.json is for elements that only need this feature.

Best Practice

The "name" needs to be different than the "id". In another word, DO NOT do this: "id": 'button1', "name": 'button1'. Otherwise you’d be wondering why certain elements are not editable and may lose a few hairs in the process.

Containers

Containers are elements that … well, contain other elements :) You can use a container by giving it the “container” type in meta/template.json 


<div data-lead-id="ex01-container" class="well">         
    ...
</div>

Text

The “text” type meta/template.json designates the enclosed text as editable. A limited number of formatting options are available for the enclosed text (i.e. bold, italic, underlined, foreground and background colors, links, and the font.)


<p data-lead-id="s01-lead" >         
    ...
</p>

Images

The image type in meta/template.json designates an image as swappable. Setting the max-width property will constrain the image dimensions, so use this with caution. That behavior is best handled in the stylesheets. 


    <img src="img/340x120.png" data-lead-id="ex03-image" alt="" class="thumbnail">

Background Images

The background-image in meta/template.json designates an image as swappable. The only difference between background-image & image types is that a background-image will remove the editable hover-over in the RapidCMS page builder. See more tips below. 


    <img src="img/340x120.png" data-lead-id="background-image" alt="" class="thumbnail">

Videos

The “video” type meta/template.json specifies a container within which the editor can embed a video. The editor will insert a placeholder during editing but including a mock image will help during the editing phase, prior to upload.

The only difference between the video element and the embed element is that the video element will handle responsive video for you.


<div  data-lead-id="ex05-video"> 
    <img src="img/videoPlaceholder.png" />
</div>

Text Link

The “link” type meta/template.json designates a hyperlink that can be edited. The “text” type should be used when links are amongst a larger body of text.


<a href="http://example.com/RapidCMS"  data-lead-id="ex06-image-link"> 
   Example
</a>

Image Link

The “image-link” type meta/template.json is a mash-up of the “image” and the “link” types. It allows image swapping and assigning a destination URL.


<a href="http://example.com/RapidCMS"  data-lead-id="ex06-image-link"> 
    <img src="img/340x120.png" alt=" "/>
</a>
